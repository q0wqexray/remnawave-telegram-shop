# Локальная сборка и запуск проекта

Это руководство поможет вам настроить и запустить Telegram-бота на вашем локальном сервере с использованием Docker.

## Требования

*   **Docker** и **Docker Compose**: [Инструкция по установке](https://docs.docker.com/get-docker/)
*   **Git**: для клонирования репозитория.
*   **Go**: версия `1.25.3` или выше (если вы решите собирать проект без Docker).

## 1. Клонирование репозитория

Сначала клонируйте репозиторий на ваш сервер. По умолчанию будет клонирована основная ветка (`main`):

```bash
git clone https://github.com/q0wqexray/remnawave-telegram-shop.git /opt/remnawave-telegram-shop-dev
cd [новое-имя-папки]
```

Для клонирования конкретной ветки (например, `dev`) используйте следующую команду:

```bash
git clone -b dev https://github.com/q0wqexray/remnawave-telegram-shop.git /opt/remnawave-telegram-shop-dev
cd [новое-имя-папки]
```

> **Примечание:** Параметр `[новое-имя-папки]` позволяет избежать конфликта имен, если папка с таким же названием, как у репозитория, уже существует на вашем компьютере.

## 2. Настройка переменных окружения

Для работы приложения необходимо задать переменные окружения. Создайте файл `.env` в корне проекта, скопировав содержимое из `.env.sample`:

```bash
cp .env.sample .env
```

Затем откройте файл `.env` и отредактируйте его, указав ваши реальные данные.

**Обязательные переменные:**

*   `TELEGRAM_TOKEN`: Токен вашего Telegram-бота.
*   `ADMIN_TELEGRAM_ID`: ID администратора в Telegram.
*   `REMNAWAVE_URL`: URL вашего Remnawave API.
*   `REMNAWAVE_TOKEN`: Токен для доступа к Remnawave API.

**Настройки базы данных:**

Переменные `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB` уже предзаполнены. **Не меняйте их**, если не знаете, что делаете, так как они используются для связи между контейнерами `bot` и `db`.

**Настройки платежных систем:**

*   `YOOKASA_ENABLED`: `true` или `false`.
*   `YOOKASA_SHOP_ID`: ID вашего магазина в ЮKassa.
*   `YOOKASA_SECRET_KEY`: Секретный ключ вашего магазина в ЮKassa.
*   `CRYPTO_PAY_ENABLED`: `true` или `false`.
*   `CRYPTO_PAY_TOKEN`: Токен вашего CryptoBot.

Заполните остальные переменные в соответствии с вашими потребностями.

## 3. Сборка и запуск

Проект использует `docker-compose` для оркестрации сервисов. Для локальной разработки рекомендуется собрать образ, чтобы подтянуть все зависимости.

### Шаг 3.1: Сборка образа (локально)

Чтобы избежать скачивания образа из `ghcr.io`, вы можете собрать его локально. Для этого закомментируйте или удалите строку `image: ghcr.io/jolymmiels/remnawave-telegram-shop-bot:latest` в файле `docker-compose.yaml` и добавьте `build: .`

```yaml
services:
  bot:
    # image: ghcr.io/jolymmiels/remnawave-telegram-shop-bot:latest
    build: . # Добавьте эту строку
    env_file:
      - .env
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - ./translations:/translations
# ... остальная часть файла
```

### Шаг 3.2: Запуск контейнеров

Выполните следующую команду в корневой директории проекта:

```bash
docker compose up --build -d
```

*   `--build`: эта опция заставит Docker пересобрать образ бота с учетом ваших локальных изменений.
*   `-d`: запустит контейнеры в фоновом режиме.

После выполнения команды будут запущены два контейнера:
1.  `remnawave-telegram-shop-bot-dev`: сам бот.
2.  `remnawave-telegram-shop-db-dev`: база данных PostgreSQL.

## 4. Проверка статуса

Чтобы убедиться, что все работает, проверьте логи контейнеров:

```bash
docker compose logs -f bot
```

Если все настроено правильно, вы увидите логи, свидетельствующие об успешном запуске бота и подключении к базе данных.

```bash
docker compose down
```
